# SupplyPro 供应链管理系统 - 系统使用说明书

## 一、系统概述

### 1.1 系统简介
SupplyPro 是一套完整的供应链管理系统，覆盖从供应商管理、商品采购、仓储库存到财务结算的全流程业务。系统采用前后端分离架构，后端基于 Spring Boot + JPA + MySQL，前端基于 React + TypeScript + Ant Design。

### 1.2 核心业务模块
| 模块 | 功能描述 |
|------|----------|
| 基础数据管理 | 品牌、供应商、物流供应商、银行、税务分类、商品分类、商品池、组合商品 |
| 采购管理 | 平台订单确认、采购单创建/发货/收货/入库、成本调价、发货单导出/导入 |
| 退款管理 | 退款单创建/确认收货（仅退款+退货退款） |
| 仓储管理 | 分仓管理、采购入库、仓库出库、库存变动记录、库存报表 |
| 结算管理 | 待结算采购单、待结算配送单、供应商结算单、预付款审批、成本票管理 |
| 物流管理 | 物流公司、物流供应商、物流轨迹查询、物流状态同步 |
| 系统维护 | 数据清理、数据修复、缓存管理（仅管理员） |

### 1.3 系统访问
- **前端地址**: http://localhost:5173/
- **后端API**: http://localhost:8080/api/
- **默认账号**: admin / 123456

## 二、基础数据管理

### 2.1 品牌管理
**路径**: 供应链管理 → 品牌管理

#### 功能说明
- **品牌列表**: 展示所有品牌信息，支持按名称、首字母、状态筛选
- **新增品牌**: 填写品牌名称、商标号、上传品牌图标
- **编辑品牌**: 修改品牌基本信息
- **关联供应商**: 为品牌添加或移除关联的供应商（多对多关系）
- **品牌状态管理**: 启用/禁用品牌

#### 品牌状态
| 状态 | 中文含义 |
|------|----------|
| ENABLED | 已启用 |
| DISABLED | 已禁用 |

#### 校验规则
| 规则 | 错误提示 | 说明 |
|------|----------|------|
| 品牌名称唯一 | 名称重复提示 | 不允许创建同名品牌 |
| 首字母自动提取 | 无（自动生成） | 根据品牌名称自动生成首字母索引 |
| 图标格式限制 | 格式错误提示 | 支持jpg/jpeg/png格式上传 |
| 品牌名称必填 | "请输入品牌名称" | 前端表单校验 |

#### 操作步骤
1. 点击「品牌管理」进入列表页
2. 点击「新建」按钮填写品牌信息
3. 在品牌详情中点击「关联供应商」按钮选择供应商
4. 支持批量操作：启用/禁用品牌状态

## 三、采购管理

### 3.1 平台订单采购确认
**路径**: 供应链管理 → 平台订单采购确认

#### 业务背景
运营平台推送的待确认采购订单需要在此确认后才能生成正式采购单。

#### 订单类型
| 类型 | 中文含义 |
|------|----------|
| OrderPurchase | 订单采购 |
| Replenishment | 补货采购 |

#### 成本承担方
| 承担方 | 中文含义 |
|------|----------|
| Platform | 平台承担 |
| Supplier | 供应商承担 |

#### 平台订单状态
| 状态 | 中文含义 |
|------|----------|
| PENDING | 待确认 |
| CONFIRMED | 已确认 |
| REJECTED | 已驳回 |

#### 校验规则
| 规则 | 错误提示 | 说明 |
|------|----------|------|
| 必须找到有效供应商 | "No valid supplier found for platform confirmation" | 先按ID找，再按名称找，最后取第一个 |
| 商品必须存在 | "Product not found: XXX" | 平台订单中的商品必须在商品池中 |
| 成本承担方决定结算状态 | 无（自动流转） | 供应商承担→已结算，平台承担→未结算 |
| 成本价必须有效 | "成本价必须为有效数字且大于等于0" | 前端校验 |
| 发货仓库和批次必须选择 | "请完整选择发货仓库和批次" | 前端校验 |
| 批次可发数量必须充足 | "所选批次可发数量不足，还差 X 件" | 前端校验 |

#### 操作流程
1. 查看待确认订单列表
2. 点击「确认」生成对应类型的采购单
3. 或点击「驳回」拒绝该订单

#### 采购单编号规则
- **格式**: C+yyyyMMddHHmm+3位序号（如C202603201536001）
- **序号上限**: 每分钟最多999个
- **唯一性保证**: 最多重试50次生成唯一编号

#### 采购单生命周期操作

##### 创建采购单
- 从平台确认订单转化
- 手动创建通用采购单（初始状态为待处理）
- 生成入库采购单（入库采购专用）

##### 创建采购单校验规则
| 规则 | 错误提示 | 说明 |
|------|----------|------|
| 至少一个商品明细 | "Purchase Order must contain at least one product item." | 不允许空采购单 |
| 防重复提交 | "请勿重复提交" | Redis幂等键5秒防重复 |
| 入库采购类型固定 | 无（强制赋值） | 入库采购单类型不可更改 |

##### 收货操作
1. 采购单到达后执行收货确认
2. 收货后自动触发入库流程：
   - 创建入库单
   - 创建采购单快照
   - 同步更新结算状态
   - 同步更新关联配送单状态为"已收货"
3. 收货失败时整个事务回滚（采购单状态不变）

##### 修改物流信息
1. 已发货/已收货状态可修改物流信息
2. **重要**：已收货状态修改物流信息后，采购单状态将回退为"已发货"
3. 运费已结算的采购单不可修改物流信息
4. 修改物流信息时使用乐观锁(@Version)保护并发修改
5. **运单号"从单"逻辑**：如果新输入的运单号在系统中已有历史记录且该历史运单号有物流费用，系统会自动将当前采购单设为该运单号的"从单"，复制历史物流信息，并将当前运费设为0

##### 发货自动创建配送单
1. 物流费用>0时自动创建待结算配送单
2. **同运单号合并逻辑**：如果同一物流供应商已有相同运单号的待结算配送单，系统会自动将当前运费累加到已有配送单中，而非创建新配送单

## 十、业务流转说明

### 10.1 采购业务主流程

#### 流转图
```
运营平台推送订单 → 平台待确认订单列表 → 确认后生成采购单(待处理)
→ 发货(已发货) → 收货(已收货) → 自动创建入库单
→ 确认入库(创建库存批次+库存流水) → 出现在待结算列表
→ 发起结算(创建结算单) → 确认结算 → 付款结算
```

#### 关键逻辑
1. **平台订单确认**：供应商匹配优先级为 ID精确匹配 → 名称模糊匹配 → 取第一个可用供应商；成本承担方决定初始结算状态
2. **采购单发货**：物流配送必须选择物流供应商+物流公司+运单号；自配送必须选择物流供应商+配送员+联系电话；运单号实时查重；物流费用>0时自动创建待结算配送单
3. **采购单收货**：自动创建入库单和采购单快照；同步更新结算状态；收货失败时整个事务回滚
4. **入库确认**：创建库存批次(批次号=日期序号+供应商名)；创建入库库存流水；入库前自动检查已审批调价单并使用调价后成本
5. **采购单删除**：只有待处理/已取消状态可删除；关联入库单必须为待入库/已取消；级联删除关联数据

### 10.2 退款业务流程

#### 流转图
```
运营平台推送退款单
├── 仅退款+供应商承担 → 自动完成(已完成) → 若采购单待处理则自动取消 → 出现在待结算列表
├── 仅退款+平台承担 → 不推送至本项目(直接拒绝)
└── 退货退款
    ├── 有物流单号 → 退货中 → 确认收货 → 已完成
    └── 无物流单号 → 待处理(等待物流信息)
    确认收货后:
    ├── 采购单+供应商承担 → 不创建入库单，退款金额记入待结算列表
    ├── 采购单+平台承担 → 创建退货入库单
    └── 出库单(不论承担方) → 创建退货入库单
```

#### 关键逻辑
1. **退款单创建**：仅退款+平台承担直接拒绝；仅退款+供应商承担自动完成并可能取消关联采购单；退货退款根据物流单号决定初始状态
2. **退款确认收货**：只有退货中状态可确认；根据业务类型和承担方决定是否创建入库单
3. **退款与结算关联**：退款金额为负数冲减采购金额；撤销结算单后退款单重新回到待结算列表

### 10.3 出库业务流程

#### 流转图
```
创建出库单 → 冻结库存(可用数量↓，冻结数量↑)
→ 发货(冻结数量↓，总数量↓，创建出库流水)
→ 签收 → 出现在待结算配送列表 → 发起物流结算
```

#### 关键逻辑
1. **出库单创建**：选择商品和批次，指定出库数量；创建时冻结库存；可发数量=可用数量-冻结数量
2. **出库单发货**：必须选择物流供应商；发货时扣减库存；批次可用数量为0时自动标记为已售罄
3. **出库单取消**：只有待发货状态可取消；取消时释放冻结库存
4. **出库单来源**：销售出库(关联销售订单)、代发出库(一件代发)、采购出库(退货出库)

### 10.4 结算业务流程

#### 流转图
```
待结算采购单列表(已收货采购单+已完成退款单)
→ 勾选+发起结算 → 创建结算单(待结算)
→ 确认 → 已结算 → 付款 → 已付款
→ 撤销 → 已撤回(项目回到待结算列表)
→ 驳回 → 已驳回
```

#### 关键逻辑
1. **待结算列表**：已收货且未结算的采购单 + 已完成且未结算的退款单；排除已纳入有效结算单的项目
2. **结算单创建**：同一供应商的采购单和退款单可合并结算；退款金额冲减采购金额；创建后已勾选项从待结算列表移除
3. **结算单付款**：支持预付款余额抵扣；余额不足时拒绝付款
4. **结算单撤销**：原结算项重新回到待结算列表；采购单和退款单结算状态重置为未结算
5. **结算单编辑/删除**：只有待结算/已撤回/已驳回可编辑；已结算/已付款不可删除；只有待结算可删除

### 10.5 预付款业务流程

#### 流转图
```
提交充值申请(待审批) → 审批通过(已审批) → 财务付款(已付款) → 余额增加
                        ↓
                     驳回(已驳回) / 撤回(已撤回)

成本票管理:
上传成本票 → 部分上传/全部上传 → 红冲票(金额≤已上传金额)
```

#### 关键逻辑
1. **预付款充值**：审批通过后由财务付款；付款后余额增加并创建充值流水
2. **预付款扣减**：结算单使用预付款余额支付时自动扣减；余额不足拒绝；扣减后创建消费流水
3. **成本票管理**：上传金额不能大于未收金额；红冲金额不能大于已上传成本票金额；状态自动流转(未上传→部分上传→已上传)
4. **预付款退回**：退回金额必须为正数；退回后余额增加并创建退回流水

### 10.6 成本调价业务流程

#### 流转图
```
对采购单发起调价 → 调价单(待审批)
→ 审批通过 → 更新采购单成本+更新库存批次成本+创建调价流水+自动创建结算单
→ 驳回 → 采购单成本不变
→ 撤销 → 采购单成本不变
```

#### 关键逻辑
1. **调价单创建**：同一采购单不能存在待审批的调价单；调价后成本不能小于0；调价前成本必须与当前一致
2. **调价审批通过**：更新采购单明细成本；入库采购类型同步更新库存批次成本；创建调价类型库存流水(仅改变成本不改变数量)；差额不为0时自动创建结算单

### 10.7 库存变动逻辑汇总

| 触发事件 | 流水类型 | 数量变动 | 成本变动 | 关联单据 |
|---------|---------|---------|---------|---------|
| 入库确认 | 入库 | +数量 | +总成本 | 入库单号 |
| 出库发货 | 出库 | -数量 | -总成本 | 出库单号 |
| 退款退货入库 | 退货入库 | +数量 | +总成本 | 退款单号 |
| 成本调价审批 | 调价 | 不变 | ±成本变动 | 调价单号 |
| 库存盘点盘盈 | 盘盈 | +数量 | +总成本 | - |
| 库存盘点盘亏 | 盘亏 | -数量 | -总成本 | - |

### 10.8 状态机汇总

| 业务对象 | 初始状态 | 可流转状态 | 终态 |
|---------|---------|-----------|------|
| 采购单 | 待处理 | 待处理→已确认→已发货→已收货→待结算 | 已取消 |

| 出库单 | 待发货 | 待发货→已发货→已签收→已完成 | 已取消 |

## 十一、常见操作指引

### 11.1 典型采购流程
```
1. 平台推送待确认订单 → 2. 确认订单生成采购单 → 3. 采购单发货
→ 4. 采购单收货 → 5. 创建入库单 → 6. 确认入库(生成库存批次)
→ 7. 采购单出现在待结算列表 → 8. 发起结算 → 9. 确认并付款
```

### 11.2 典型退款流程
```
1. 运营平台推送退款单 → 2. 仅退款+供应商: 自动完成
                         2. 退货退款: 填写物流信息 → 3. 状态变为退货中
→ 4. 退货运回仓库 → 5. 确认收货 → 6. 生成退货入库单(如果是平台承担或出库单)
→ 7. 退款单出现在待结算列表 → 8. 发起结算
```

### 11.3 典型出库流程
```
1. 创建出库单(选择商品批次) → 2. 冻结库存 → 3. 填写物流信息并发货
→ 4. 扣减实际库存 → 5. 客户签收 → 6. 出库单出现在待结算配送列表
→ 7. 发起物流结算
```

### 11.4 典型预付款流程
```
1. 提交充值申请 → 2. 审批通过 → 3. 财务付款 → 4. 预付款余额增加
→ 5. 结算时使用预付款余额抵扣 → 6. 上传成本票 → 7. 成本票审核
```
