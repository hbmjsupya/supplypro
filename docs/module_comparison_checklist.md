# 模块间代码对比检查清单 (Module Comparison Checklist)

为防止将供应商管理模块（Benchmark）的功能迁移到物流供应商模块时出现遗漏或不一致，请在开发和代码审查阶段对照以下清单执行。

## 1. 数据模型 (Data Model)
- [ ] **字段对齐**：核心业务字段（如 `purchaser_id`, `purchaser_name`, `settlement_cycle`）名称和类型是否一致？
- [ ] **关联关系**：一对多、多对多关系（如文件、账户、联系人）的级联策略 (`CascadeType`, `orphanRemoval`) 是否一致？
- [ ] **审计字段**：是否都包含了 `created_by`, `created_time`, `updated_by`, `updated_time`？
- [ ] **唯一性约束**：是否在数据库层面实施了相同的唯一性索引（如统一社会信用代码）？

## 2. API 接口 (API & DTO)
- [ ] **路由规范**：URL 结构是否统一（例如 `/api/{module}/{id}/files`）？
- [ ] **DTO 字段**：返回给前端的 DTO 是否包含相同的字段别名（如 `procurementOwner` vs `purchaserName`）？
- [ ] **分页与排序**：列表接口的分页参数 (`page`, `size`) 和默认排序规则是否一致？
- [ ] **错误码**：业务异常（如“名称已存在”）返回的错误码和 HTTP 状态码是否统一？

## 3. 前端实现 (Frontend Implementation)
- [ ] **组件复用**：是否复用了通用的业务组件（如 `SupplierFileManager`, `SettlementCycleSelect`）而不是复制粘贴代码？
- [ ] **表单校验**：必填项校验、正则规则（如手机号、邮箱）是否一致？
- [ ] **交互体验**：
    - [ ] 列表页是否都有搜索、排序、分页？
    - [ ] 删除操作是否都有“二次确认”弹窗？
    - [ ] 详情页布局（Tab 页签、区块顺序）是否一致？
- [ ] **状态管理**：数据加载 (`loading` 状态) 和错误提示 (`message.error`) 逻辑是否一致？

## 4. 文件服务 (File Service)
- [ ] **配置一致性**：上传大小限制、允许的文件类型列表是否一致？
- [ ] **存储策略**：是否都使用了相同的存储根目录结构（按模块/按日期分文件夹）？
- [ ] **临时文件处理**：新增实体时的临时文件上传、绑定、清理逻辑是否一致？
- [ ] **下载鉴权**：文件下载接口的权限控制（Token 校验）是否一致？

## 5. 安全与权限 (Security & Permissions)
- [ ] **数据权限**：列表查询是否都应用了相同的数据权限过滤（如仅看自己负责的供应商）？
- [ ] **功能权限**：增删改查按钮的权限标识（Permission Key）命名规范是否统一？
