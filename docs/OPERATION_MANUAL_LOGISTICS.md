# 物流与入库单管理操作手册

## 1. 概述
本手册旨在指导用户使用供应链系统的物流管理与入库单状态同步功能。涵盖采购单发货、入库单自动生成、物流信息追踪及异常处理流程。

## 2. 状态流转说明

### 2.1 采购单状态流转
- **新建 (PENDING)**: 采购单创建后的初始状态。
- **待确认 (PENDING)** -> **已确认 (CONFIRMED)**: 审核通过。
- **已确认 (CONFIRMED)** -> **已生成入库单 (GENERATED_INBOUND)**: 
  - 对于 `INBOUND` 类型的采购单，系统会自动生成关联的入库单。
  - **注意**: 此状态节点已恢复，用于明确标识单据生命周期。
- **已生成入库单** -> **已发货 (SHIPPED)**: 
  - 供应商或物流专员在系统中录入物流信息后，状态自动更新。
  - 此时入库单状态保持不变，但会同步显示物流信息。
- **已发货** -> **已入库 (RECEIVED)**: 仓库确认收货。
- **已入库** -> **已完成 (COMPLETED)**: 财务结算完成。

### 2.2 状态同步机制
- **发货同步**: 采购单更新物流信息时，系统自动记录操作日志，并更新采购单发货状态为 `SHIPPED`。
- **取消同步**: 若采购单被取消，关联的入库单将自动同步为 `CANCELLED` 状态。
- **异常监控**: 系统每10分钟自动扫描状态不一致的单据（如采购单已取消但入库单未取消），并生成异常日志。

## 3. 功能操作指南

### 3.1 新建入库采购单 (自动填充)
1. 进入 **采购管理 -> 新建采购单**。
2. 选择 **入库采购** 类型。
3. 选择 **入库仓库**。
   - 系统将自动填充该仓库的 **收货地址** 和 **联系人信息**（基于仓库管理员配置）。
   - 用户可手动修改联系人电话或详细地址。
4. 提交后，系统自动生成入库单。

### 3.2 录入物流信息
1. 进入 **采购单列表**，找到状态为 `GENERATED_INBOUND` 或 `CONFIRMED` 的单据。
2. 点击 **发货/更新物流** 按钮。
3. 输入物流公司（如：顺丰速运）和运单号。
4. 确认后，采购单状态变为 **已发货**，入库单详情页将同步显示物流追踪信息。

### 3.3 查看入库单物流详情
1. 进入 **入库单列表**。
2. 点击 **详情**。
3. 在详情页中查看新增的 **物流信息** 模块，包含：
   - 物流公司、运单号
   - 发货时间
   - 物流轨迹（支持点击展开）

### 3.4 入库单列表管理
1. **列表展示**：
   - 显示所有关联采购单的入库单据。
   - 支持按入库单号、采购单号、供应商等条件筛选。
2. **发货状态列**：
   - 展示关联采购单的物流状态（未发货/待发货/已发货/已收货）。
   - **注意**：该列仅作信息展示，不支持排序和筛选功能。

### 3.5 异常数据诊断
1. 若发现单据状态异常，可联系管理员访问诊断接口：
   - `/api/diagnostic/check-logistics`: 检查并返回状态不一致的单据列表。

## 4. 字段说明

| 字段名 | 说明 | 格式/示例 |
| :--- | :--- | :--- |
| **下单时间** | 采购单生成的精确时间 | YYYY-MM-DD HH:mm:ss |
| **发货状态** | 独立于业务状态的物流状态 | 待发货(灰色)/已发货(蓝色)/已签收(绿色) |
| **入库单号** | 自动生成的唯一单号 | IN + 日期 + 序列号 (如 IN202310271430001) |
| **收货地址** | 仓库物理地址 | 省/市/区 + 详细地址 |

## 5. 常见问题
- **Q: 为什么新建采购单后列表看不到？**
  - A: 请检查是否选择了正确的仓库，且该仓库已配置管理员权限。
- **Q: 入库单为何没有物流信息？**
  - A: 需在采购单侧点击"发货"并录入运单号，入库单才会同步显示。
