# SupplyPro 供应链管理系统 - 测试用例

## 一、业务流转测试用例

### 1.1 采购业务主流程

#### TC-FLOW-001 标准采购完整流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在供应商、商品、仓库 |
| 测试步骤 | 1. 在平台订单确认页面确认一条订单采购类型的待确认订单<br>2. 系统自动生成采购单，状态为"待处理"<br>3. 对采购单执行发货操作，选择物流公司和运单号<br>4. 采购单状态变为"已发货"<br>5. 对采购单执行收货操作<br>6. 采购单状态变为"已收货"，自动创建入库单<br>7. 确认入库，系统创建库存批次和入库流水<br>8. 采购单出现在待结算列表中<br>9. 勾选采购单发起结算，创建结算单<br>10. 确认结算单，状态变为"已结算"<br>11. 付款结算单，状态变为"已付款" |
| 预期结果 | 全流程状态正确流转，库存批次和流水正确生成，结算单金额正确 |
| 优先级 | P0 |

#### TC-FLOW-002 补货采购流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在供应商、商品、仓库 |
| 测试步骤 | 1. 在平台订单确认页面确认一条补货采购类型的待确认订单<br>2. 后续步骤同TC-FLOW-001步骤3-11 |
| 预期结果 | 采购单类型为"补货采购"，其余流程与标准采购一致 |
| 优先级 | P0 |

#### TC-FLOW-003 入库采购流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在供应商、商品、仓库 |
| 测试步骤 | 1. 创建入库采购单，选择供应商和仓库<br>2. 系统自动创建采购单和入库单<br>3. 确认入库，库存批次创建<br>4. 入库采购单不产生结算 |
| 预期结果 | 入库采购单类型固定为"入库采购"，自动创建入库单，不出现在待结算列表 |
| 优先级 | P0 |

### 1.2 退款业务流程

#### TC-FLOW-004 仅退款+供应商承担流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在待处理的采购单 |
| 测试步骤 | 1. 运营平台推送退款单：仅退款+供应商承担<br>2. 系统自动完成退款单，状态直接变为"已完成"<br>3. 若关联采购单为"待处理"，自动取消<br>4. 退款单出现在待结算列表 |
| 预期结果 | 退款单自动完成，不创建入库单，退款金额记入待结算列表 |
| 优先级 | P0 |

#### TC-FLOW-005 退货退款+供应商承担流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在已收货的采购单 |
| 测试步骤 | 1. 运营平台推送退款单：退货退款+供应商承担，包含物流单号<br>2. 退款单状态为"退货中"<br>3. 确认收货<br>4. 退款单状态变为"已完成"<br>5. 不创建退货入库单（采购单+供应商承担）<br>6. 退款单出现在待结算列表 |
| 预期结果 | 退款单确认收货后不创建入库单，退款金额记入待结算列表 |
| 优先级 | P0 |

#### TC-FLOW-006 退货退款+平台承担流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在已收货的采购单 |
| 测试步骤 | 1. 运营平台推送退款单：退货退款+平台承担，包含物流单号<br>2. 退款单状态为"退货中"<br>3. 确认收货<br>4. 退款单状态变为"已完成"<br>5. 自动创建退货入库单<br>6. 确认入库后创建退货入库流水(RETURN_IN) |
| 预期结果 | 退款单确认收货后创建退货入库单，库存增加 |
| 优先级 | P0 |

#### TC-FLOW-007 出库单退款流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已存在已发货的出库单 |
| 测试步骤 | 1. 运营平台推送退款单：退货退款，关联出库单<br>2. 退款单状态为"退货中"<br>3. 确认收货<br>4. 自动创建退货入库单（不论承担方）<br>5. 确认入库后创建退货入库流水 |
| 预期结果 | 出库单退款确认收货后一定创建退货入库单 |
| 优先级 | P0 |

### 1.3 出库业务流程

#### TC-FLOW-008 仓库出库完整流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 仓库中存在可用库存批次 |
| 测试步骤 | 1. 创建出库单，选择商品和批次<br>2. 系统冻结库存（可用数量减少，冻结数量增加）<br>3. 执行发货操作，填写物流信息<br>4. 系统扣减库存（冻结数量减少，总数量减少）<br>5. 出库单状态变为"已发货"<br>6. 创建出库流水(OUTBOUND)<br>7. 确认签收<br>8. 出库单出现在待结算配送列表 |
| 预期结果 | 库存冻结→扣减→流水正确，出库单状态流转正确 |
| 优先级 | P0 |

#### TC-FLOW-009 出库单取消流程
| 项目 | 内容 |
|------|------|
| 前置条件 | 已创建待发货的出库单 |

| 优先级 | P0 |

| 预期结果 | 预付款余额正确增加，流水记录正确 |
| 优先级 | P0 |

### 1.5 成本调价流程

#### TC-FLOW-013 成本调价完整流程

|------|------|
| 前置条件 | 已存在已收货的采购单，有库存批次 |
